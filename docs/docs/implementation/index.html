Implementation

Models

There are several routes in the API used in the project that return some objects. These objects are people, planets, starships, among others. You must have an equivalent C # object for each object returned by the API.

However, for routes that return more than one item, the returned object has its own structure. So there is a generic class called ModelList that is representing the returned object. 

public class ModelList<T>
{
    public int Count { get; set; }
    public string Next { get; set; }
    public string Previous { get; set; }
    public IList<T> Results { get; set; }
}

API

In order to make API queries easier, a solution was implemented that encapsulates and makes generic all calls, return treatments, object conversion.

Just make use of the class BaseApi passing the Model that will be used for callback. You must also enter which route to call on the BaseUri property

public class StarShipsApi : BaseApi<StarShip>, IStarShipsApi
{
    public StarShipsApi(IConfigurationRoot config) : base(config)
    {
        BaseUri = "starships";
    }
}

Services

In this challenge, Service was used as a business flow orchestration object. Tasks such as requesting information from the user, API requests, information processing and output of processed data are performed.

Dependency Injection

In this challenge a well-known and widely used developmental pattern of Dependency Injection was used. To perform the configuration of an injection, just register in class Startup.cs.

collection.AddScoped<IStarShipsApi, StarShipsApi>();
collection.AddScoped<IStarShipsService, StarShipsService>();
